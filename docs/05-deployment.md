# 发布与部署说明

本文记录项目当前真实可用的发布方式，以及未来如果接入服务端后的部署预留。

当前项目还没有服务端，因此不采用“云服务器 + docker-compose + nginx + GitHub Actions”作为主发布策略。此前那套方案对当前阶段并不适用，应视为未来预留，而不是现在的正式发布方式。

## 当前状态

### H5

- H5 是静态前端项目
- 当前可以本地构建后部署到任意静态托管环境
- 如果暂时没有正式站点，也可以只保留本地构建产物用于测试

### 微信小程序

- 当前没有服务端依赖
- 当前发布方式是：
  - 本地构建微信小程序包
  - 使用微信开发者工具打开产物目录
  - 通过微信公众平台 / 微信开发者工具上传代码
- 这也是当前阶段最符合实际的发布路径

## 当前推荐发布方式

### 1. H5 发布

H5 当前属于纯静态站点，推荐流程：

1. 本地执行构建
2. 得到 `dist` 产物
3. 将产物部署到静态托管环境

可选托管方式：

- Nginx 静态站点
- GitHub Pages
- Vercel / Netlify / Cloudflare Pages
- 任意对象存储静态托管

### 2. 微信小程序发布

微信小程序当前推荐流程：

1. 执行构建命令
   - `pnpm build:uni:mp-weixin`
2. 在微信开发者工具中打开目录
   - `apps/uni/dist/mp-weixin`
3. 本地预览与真机调试
4. 上传代码到微信公众平台
5. 在微信公众平台提交审核 / 发布

## 为什么当前不写服务端部署方案

因为当前项目：

- 没有后端服务
- 没有数据库
- 没有接口网关
- 没有需要常驻运行的服务进程

所以此时写完整的服务器部署设计，会出现两个问题：

- 对当前阶段无效
- 容易让文档和真实发布方式脱节

当前更合适的做法是：

- H5 按静态资源部署理解
- 小程序按微信公众平台发布理解

## 未来预留

如果后续进入二期或三期，出现以下需求，再补充服务端部署设计更合适：

- 用户账号体系
- 云端词库同步
- 对战记录 / 战绩
- 房间联机
- AI 词库服务
- 运营配置后台

到那时再引入以下内容会更合理：

- 云服务器
- Docker / docker-compose
- Nginx 反代
- GitHub Actions 自动部署
- 数据库与对象存储

## 当前发布检查清单

### H5

- 本地构建通过
- 页面静态资源可正常访问
- 路由刷新策略与部署平台匹配

### 微信小程序

- 小程序构建通过
- 微信开发者工具能正常打开 `apps/uni/dist/mp-weixin`
- 真机预览通过
- 上传到微信公众平台成功

## 结论

当前阶段应把“发布”和“部署”分开理解：

- 微信小程序：本地构建后上传到微信公众平台
- H5：构建后部署到静态托管环境

服务端部署方案暂不作为当前文档重点，等项目真正引入后端后再补充。
