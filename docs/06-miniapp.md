# 微信小程序规划

本文记录微信小程序的大版本视角规划，按阶段分为一期、二期、三期。

一期已经完成并上架；一期内的小版本优化与迭代，不再混写在这里，单独记录在 `08-miniapp-iterations.md`，便于版本管理与回溯。

## 一期（已上架）

### 目标

- 微信原生风格 UI
- 核心流程可玩：开局 → 逐个查看 → 房主查验 / 忘词回看
- 本地存储：词库、自定义词对、本局 session、verdicts
- 支持小程序分享、词对难度反馈与持续扩词

### 当前实现位置

- uni-app 工程：`apps/uni/`
  - 开局：`apps/uni/src/pages/setup/index.vue`
  - 词对：`apps/uni/src/pages/words/index.vue`
  - 发牌：`apps/uni/src/pages/reveal/index.vue`
  - 房主：`apps/uni/src/pages/host/index.vue`

### 说明

- 一期以“单机可玩、线下聚会即开即用”为目标
- 不要求联机房间、账号体系、后端同步
- 分享、推广、转化相关能力作为一期内的小版本迭代单独管理
- 当前一期小版本已推进到 `0.0.2`
- `0.0.3` 计划继续在一期内迭代“词库难度分数化与轻运营”，暂不引入完整服务端

## 二期（扩展）

### 方向

- 后端同步
  - 用户状态、战绩、配置同步
  - 建议新增 `apps/uni/src/services/api.ts` 或独立 `packages/api`
- AI 词库辅助
  - 主题选择
  - 生成候选词对
  - 人工确认后一键入库

### 目标

- 从“可玩工具”升级为“可持续使用的游戏产品”
- 让词库管理和内容增长更轻松

## 三期（长期）

### 方向

- 更完整的游戏规则支持
- 多轮战绩与数据沉淀
- 玩法扩展、主题化活动、增长机制

### 目标

- 从线下发牌器扩展到更完整的聚会游戏产品形态

## 维护建议

当前 `apps/uni/` 内为独立实现，便于快速上线与维护。当准备长期演进时，建议：

1. 抽出纯 TS 核心逻辑到 `packages/core`
2. 抽存储适配到 `packages/storage`
3. 让 H5 与小程序复用 core / storage，只保留各自 UI 与路由
